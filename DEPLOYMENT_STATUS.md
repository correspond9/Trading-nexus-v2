## Root Cause Analysis

**Discovery:**
1. The `/api/v2/instruments/search` endpoint returns a 404 with Coolify error message
2. Traefik logs show: `"error while parsing rule Host(``) && PathPrefix(`api.tradingnexus.pro`)"`
3. The http-0 Traefik router has a malformed rule with empty `Host()` matcher
4. This prevents Traefik from properly routing requests to the backend

**Why It Happened:**
- When Coolify rebuilds the docker-compose.yaml, it auto-generates routing labels
- The generated labels contained broken syntax with `Host(``)` (empty host matcher)
- Traefik cannot parse these rules, causing routing to fail
- All requests return 404 instead of being forwarded to the backend

**Technical Details:**
```yaml
# BROKEN (in current docker-compose.yaml):
- traefik.http.routers.http-0-p488ok8g8swo4ockks040ccg-backend.rule=Host(``) && PathPrefix(`api.tradingnexus.pro`)

# ERROR: "empty args for matcher Host, []"
# This rule cannot match any request because Host() is empty

# WORKING RULE (already exists):
- traefik.http.routers.tradingbackend.rule=Host(`api.tradingnexus.pro`)
# This rule works - it matches requests for api.tradingnexus.pro domain
```

## The Fix

**Remove the broken Traefik labels** from `/data/coolify/applications/p488ok8g8swo4ockks040ccg/docker-compose.yaml`

Specifically, remove any lines containing:
- `traefik.http.routers.http-0-*`
- `traefik.http.middlewares.http-0-*`  
- `caddy_*`
- `caddy_ingress_network`

Then restart:
```bash
cd /data/coolify/applications/p488ok8g8swo4ockks040ccg
docker compose down --remove-orphans
docker compose up -d
```

**See [TRAEFIK_FIX_MANUAL_GUIDE.md](./TRAEFIK_FIX_MANUAL_GUIDE.md) for complete step-by-step instructions.**
